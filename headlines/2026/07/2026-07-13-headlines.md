# AI Headlines — 2026-07-13

- [What xAI Grok Build CLI actually sends to xAI — a wire-level analysis](https://gist.github.com/cereblab/dc9a40bc26120f4540e4e09b75ffb547) — Researcher spliced a logging proxy between Grok Build CLI 0.2.93 and the model endpoint; found the CLI uploads the entire repository as a git bundle (~5.1 GiB from a 12 GB repo) to `gs://grok-code-session-traces` on Google Cloud Storage — including files the agent never read and `.env` credentials — alongside 192 KB of actual model context; the "Improve the model" toggle does not stop the upload. *(July 13, 2026)*

- [Flash-MSA: open-source training kernels for MiniMax Sparse Attention](https://nanduruganesh.github.io/flash-msa/) — First public performant training kernels for the MSA block-sparse attention mechanism (previously open only at inference), implemented in CuTeDSL for Hopper and Blackwell GPUs; introduces group-specialized proxy heads that let different attention-head subsets select different token subsets, with speedups over standard FlashAttention growing sharply above 512k-token sequences. *(July 11, 2026)*

- [Claude Code Sends 4.7x More Tokens Than OpenCode Before Reading Your Prompt](https://systima.ai/blog/claude-code-vs-opencode-token-overhead) — Wire-level comparison (logging proxy, SHA-256 audit trail) finds Claude Code 2.1.207 sends ~32,800 tokens of scaffolding per turn vs. OpenCode's ~6,900 on Sonnet 4.5; OpenCode's prefix is byte-identical across sessions enabling efficient caching, while Claude Code rewrites cache mid-session; on multi-step tasks Claude Code's batching of tool calls can reverse the total-cost gap. *(July 12, 2026)*

- [I love LLMs, I hate hype](https://geohot.github.io/blog/jekyll/update/2026/07/12/i-love-llms.html) — George Hotz argues that LLMs are genuinely useful productivity tools — but that frontier labs deliberately amplify existential panic ("window closing," "underclass") to justify massive valuations while discouraging the open-source competition that would reveal AI's commodifying trajectory; the real critique is not the technology but who captures the value. *(July 12, 2026)*
